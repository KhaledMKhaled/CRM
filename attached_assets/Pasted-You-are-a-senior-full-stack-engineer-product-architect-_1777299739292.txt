==================================================
1. APPLICATION OVERVIEW
==================================================

Build a CRM + Marketing Attribution + Sales Performance + ROAS Analytics Web App.

The application should allow the business to:

1. Import or manually enter Meta Ads data.
2. Import or manually enter prospect and lead data.
3. Track the full funnel from prospect to deal.
4. Track sales activities and follow-ups.
5. Attribute revenue back to campaigns, ad sets, and ads.
6. Calculate true performance metrics such as CPL, CPM, CPC, Cost per MQL, Cost per SQL, CAC, Revenue, ROAS, Conversion Rates, Reply Rates, and SLA performance.
7. Provide dashboards for Admin, Sales Execution, and Media Buyer users.
8. Allow configurable fields, stages, statuses, scoring rules, channels, products, campaigns, and KPIs without breaking the system.
9. Allow shared visibility between sales and media teams so both teams can understand quality, cost, and revenue performance.
10. Support future API integrations with Meta Ads, WhatsApp, Messenger, Instagram DM, website forms, and external CRMs.

4. CORE BUSINESS FLOW
==================================================

The system must support this business flow:

1. Meta Ads campaign runs on Facebook, Instagram, Messenger, WhatsApp, or website.
2. The campaign generates conversations, leads, forms, registrations, calls, or website actions.
3. Prospect information is captured manually, imported, or later through API.
4. Each prospect is linked to:
   - Campaign
   - Ad Set
   - Ad
   - Channel
   - Source
   - Date
   - Sales owner
5. Sales team follows up.
6. Activities are logged.
7. Lead is qualified as MQL if it matches marketing qualification rules.
8. Lead is qualified as SQL if sales confirms real opportunity.
9. Deal is created if there is commercial opportunity.
10. Deal is either won, lost, or moved to nurture.
11. Revenue is recorded.
12. Revenue is attributed back to the campaign/ad set/ad.
13. Dashboard calculates true ROAS and sales efficiency.

8. CALCULATION LOGIC
==================================================

Implement a calculation engine that calculates all metrics dynamically.

Use safe division. If denominator is zero, return 0 or null based on context.

Use these formulas:

--------------------------------------------------
Basic Meta Metrics
--------------------------------------------------

CTR = clicks / impressions

CPC = amount_spent / clicks

CPM = amount_spent / impressions * 1000

Cost per Conversation = amount_spent / messaging_conversations_started

Reply Rate = messaging_conversations_replied / messaging_conversations_started

Cost per Reply = amount_spent / messaging_conversations_replied

Meta Lead Rate = meta_leads / messaging_conversations_started

Website Registration Rate = website_registrations_completed / clicks

--------------------------------------------------
CRM Funnel Metrics
--------------------------------------------------

CRM Leads = count of prospects

MQL Count = count of prospects where lead_stage = MQL or later

SQL Count = count of prospects where lead_stage = SQL or later

Deals Count = count of deals

Deals Won = count of deals where deal_status = Won

Deals Lost = count of deals where deal_status = Lost

Revenue = sum of actual_revenue for won deals

Expected Revenue = sum of expected_revenue for open deals

Lead to MQL Rate = MQL Count / CRM Leads

MQL to SQL Rate = SQL Count / MQL Count

SQL to Deal Rate = Deals Count / SQL Count

Deal Win Rate = Deals Won / Deals Count

Lead to Deal Rate = Deals Won / CRM Leads

--------------------------------------------------
Cost Metrics
--------------------------------------------------

CPL = amount_spent / CRM Leads

Cost per MQL = amount_spent / MQL Count

Cost per SQL = amount_spent / SQL Count

CAC = amount_spent / Deals Won

--------------------------------------------------
Revenue Metrics
--------------------------------------------------

ROAS = Revenue / amount_spent

Profit = Revenue - amount_spent

ROI = Profit / amount_spent

Average Deal Size = Revenue / Deals Won

Revenue per Lead = Revenue / CRM Leads

Revenue per MQL = Revenue / MQL Count

Revenue per SQL = Revenue / SQL Count

--------------------------------------------------
SLA Metrics
--------------------------------------------------

First Response Time = first_reply_datetime - lead_created_datetime

SLA Met = first_response_time <= configured SLA minutes

SLA Breached = first_response_time > configured SLA minutes

SLA Compliance Rate = SLA Met Count / Total Leads

Overdue Tasks = count of tasks where due_date < now and status not completed

--------------------------------------------------
Time Grouping Logic
--------------------------------------------------

Day = date

Week = ISO week number

Month = year-month

Quarter = year-quarter

All dashboards must support grouping by:

Day
Week
Month
Quarter

Business Problem:

The company runs Meta Ads campaigns that generate messages, leads, calls, and website registrations. The marketing team can see Meta metrics, but cannot easily understand which campaigns generate real qualified leads, sales opportunities, revenue, and ROAS. The sales team talks to prospects daily but outcomes are not fully connected to campaign cost and source.

Business Objective:

Build a web application that connects media buying data with real CRM and sales outcomes to measure true marketing and sales performance.

- Every lead is linked to a source.

21. FUNCTIONAL LOGIC
==================================================

When a new prospect is created:

- Assign unique prospect code.
- Capture source/channel.
- Try to match campaign/adset/ad.
- Set initial stage as Lead.
- Create activity: Lead Created.
- Assign to sales user if provided.
- Calculate lead score based on scoring rules.
- Create first follow-up task if automation rule exists.

When a sales user updates a lead stage:

- Save old stage and new stage.
- Create activity: Stage Changed.
- Update timestamps:
  - mql_date if moved to MQL
  - sql_date if moved to SQL
  - won_date if moved to Deal Won
  - lost_date if moved to Deal Lost
- Recalculate funnel metrics.

When a deal is won:

- Mark deal_status = Won.
- Set actual revenue.
- Set won date.
- Attribute revenue to prospect’s campaign/adset/ad.
- Update revenue reports.
- Update ROAS.

When a campaign performance row is added:

- Validate date/campaign/adset/ad.
- Store spend and Meta metrics.
- Recalculate aggregated performance.
- Update dashboards.

When imported data has unmatched campaign/ad/adset:

- Create warning.
- Allow Admin to map manually.
- Do not delete data.
- Mark as unmatched until fixed.
